# 概览

## 什么是 qiankun？

qiankun 是蚂蚁集团开源的微前端解决方案，基于 single-spa 并进行了大量优化和扩展。它的名字来源于**"千里之行，始于足下"**，寓意着微前端架构的落地需要从小处着手。

## 核心特性

### 1. 样式隔离

qiankun 提供了多种样式隔离方案：

- **CSS Modules**: 通过构建工具实现样式模块化
- **Shadow DOM**: 利用浏览器原生能力进行样式隔离
- **Scoped CSS**: 通过添加特定前缀实现样式隔离

### 2. JS 沙箱

qiankun 实现了完善的 JavaScript 沙箱机制：

- **快照沙箱**: 适用于单实例微应用
- **代理沙箱**: 适用于多实例微应用
- **模块化沙箱**: 基于 ES Modules 的新一代沙箱

### 3. 资源加载

- 支持 HTML Entry 方式加载微应用
- 自动提取和执行子应用的 JS/CSS
- 支持静态资源预加载

### 4. 通信系统

- 提供全局状态管理方案
- 支持微应用之间的消息传递
- 兼容单应用的状态管理

## 架构演进

### qiankun 1.x

基于 single-spa 的简单封装，主要解决样式隔离和 JS 沙箱问题。

### qiankun 2.x

引入了更完善的沙箱机制和资源加载系统，支持多种构建工具。

### qiankun 3.x

完全重构，采用 Monorepo 架构，改进沙箱实现，支持更多现代特性。

## 核心模块

```
qiankun/
├── packages/qiankun      # 核心运行时
├── packages/sandbox      # 沙箱实现
├── packages/loader       # 资源加载器
├── packages/shared       # 共享工具函数
├── packages/ui-bindings  # UI 框架绑定
└── packages/create-qiankun # CLI 工具
```

## 适用场景

- ✅ 需要将多个独立应用整合到一个平台
- ✅ 不同团队使用不同技术栈（React、Vue、Angular 等）
- ✅ 需要独立开发、部署、运行子应用
- ✅ 需要动态加载和卸载微应用

## 不适用场景

- ❌ 简单的单页应用
- ❌ 所有子应用使用相同技术栈
- ❌ 对性能要求极其严苛的场景
- ❌ 需要支持非常古老的浏览器
